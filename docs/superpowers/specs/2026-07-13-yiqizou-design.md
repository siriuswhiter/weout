# 一起走 — 出行协作 Todo 小程序设计文档

> 日期: 2026-07-13
> 状态: 设计确认

## 1. 产品定位

"一起走"是一款微信小程序，解决朋友约出门时"谁带什么、谁负责什么"容易忘记和信息不同步的问题。用户创建一个行程后，邀请同行的朋友加入，共同维护一份 Todo 清单，每个待办可以指派给具体的人或标记为所有人都要做的事。信息近实时同步，关键变更通过微信订阅消息通知。

对于特定目的地（如海边、露营、滑雪），需要准备的东西大同小异，因此提供"广场"功能，用户可以把自己的 Todo 清单发布为模板供他人一键复用。

## 2. 目标用户

- 3-10 人的朋友/家庭出行小团体
- 场景覆盖从周末短途到长途旅行，MVP 以短途为主

## 3. 核心功能

### 3.1 行程管理

- **创建行程**: 填写行程名称、目的地（可选）、出行日期
- **邀请成员**: 两种方式
  - 分享小程序卡片到微信群/私聊，对方点击直接加入
  - 生成邀请码/二维码，对方输入或扫码加入
- **行程列表**: 按"进行中"和"已结束"分组展示，卡片显示行程名称、日期、完成进度、成员头像
- **行程状态**: 出行日期过后自动归入"已结束"，也可手动归档

### 3.2 协作 Todo

- **添加 Todo**: 底部弹窗，输入内容 → 选择指派人 → 可选截止时间 → 可选备注 → 确认
- **指派**: 每个 Todo 可标记为"所有人"或指派给一个/多个具体成员
- **截止时间**: 可选，设置后快到期时触发订阅消息提醒指派人
- **完成**: 点击圆圈即可勾选完成，已完成项显示删除线
- **筛选**: 支持按"全部 / 我的 / 未完成"筛选查看
- **操作**: 左滑删除，点击编辑

### 3.3 广场（模板社区）

- **发布模板**: 用户可将自己行程的 Todo 清单一键发布为模板
- **可见度**: 三级设置
  - 公开 — 出现在广场，所有人可见
  - 链接可见 — 不出现在广场，有链接才能访问
  - 私有 — 仅自己可见
- **浏览**: 按标签分类（海边、露营、滑雪、出国等），支持搜索
- **互动**: 点赞 + 使用次数统计，无评论功能
- **复用**: 一键将模板中的 Todo 项复制到自己的行程中

### 3.4 同步与通知

- **轮询同步**: 页面打开时每 10-30 秒自动拉取最新数据
- **订阅消息**: 关键变更（新增任务、有人完成了分配给你的任务、截止时间临近）通过微信订阅消息通知
- **授权机制**: 用户在行程页主动点击"订阅通知"按钮授权

## 4. 页面结构

### 4.1 Tab 结构（3 个底部 Tab）

| Tab | 名称 | 说明 |
|-----|------|------|
| 1 | 行程 | 我的行程列表，入口页 |
| 2 | 广场 | 模板社区，浏览和搜索模板 |
| 3 | 我的 | 个人信息、我发布的模板、设置 |

### 4.2 页面清单

1. **行程列表页** — 进行中/已结束的行程卡片列表，右上角"+"创建新行程
2. **创建行程页** — 填写行程名称、目的地、日期
3. **行程详情页** — 行程信息 + 成员列表 + Todo 清单 + 筛选 + 底部操作栏（添加 Todo / 用模板）
4. **添加 Todo 弹窗** — 底部半屏弹窗，输入内容、指派人、截止时间、备注
5. **邀请成员页** — 展示邀请码/二维码，分享按钮
6. **广场首页** — 搜索栏 + 标签筛选 + 模板卡片列表
7. **模板详情页** — 模板内容预览 + 一键使用按钮
8. **发布模板页** — 选择行程 → 设置模板名称、标签、可见度 → 发布
9. **我的页面** — 头像昵称、我发布的模板列表、设置入口

| _id | string | 用户 ID |

| completedBy | string | 完成人 ID |
| completedAt | timestamp | 完成时间 |
| creatorId | string | 创建者 ID |
| createdAt | timestamp | 创建时间 |

### 5.4 模板 (templates)

| 字段 | 类型 | 说明 |

| name | string | 模板名称 |

| visibility | string | public / link / private |

| likeCount | number | 点赞数 |
| useCount | number | 使用次数 |

### 5.5 点赞记录 (likes)

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 记录 ID |

## 6. 技术方案

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | 微信小程序原生 | 轻量、无需额外框架 |
| 后端 | 微信云开发 | 云函数 + 云数据库，免运维 |
| 登录 | 微信登录 | wx.login 获取 openId，无需额外注册流程 |
| 同步 | 轮询 + 订阅消息 | 页面内轮询 10-30s，关键变更推送订阅消息 |
| 存储 | 云数据库 | NoSQL 文档数据库，适合此类结构 |

## 7. 交互要点

- **操作简洁**: 添加 Todo 用底部弹窗而非跳转新页面，减少层级
- **一键操作**: 点击圆圈完成 Todo，左滑删除，点击编辑
- **筛选便捷**: 行程详情页顶部 Tab 快速切换全部/我的/未完成
- **邀请简单**: 分享小程序卡片到微信群，对方点击即加入
- **模板复用**: 广场中一键使用，Todo 项自动复制到当前行程

## 8. MVP 范围

### 包含
- 行程 CRUD + 成员邀请（小程序卡片 + 邀请码）
- Todo CRUD + 指派 + 可选截止时间 + 完成勾选
- 轮询同步
- 广场：浏览、搜索、标签筛选、点赞、一键使用
- 模板发布（三级可见度）
- 微信登录

### 不包含（后续迭代）
- 订阅消息通知（需要申请模板，MVP 后接入）
- 模板评论
- 行程聊天
- 费用 AA 分摊
